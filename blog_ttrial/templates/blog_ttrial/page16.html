As seen above, the result remains unchanged even after adding a margin.

This is due to a phenomenom in CSS called collapsing margins. In CSS, vertical margins of block-level elements can collapse. This means that, if you have a <p> element followed by an <a> element, the bottom margin of the <p> element and the top margin of the <a> can combine into a single margin. As a result, the margin-top on the <a> element might not create the espected space.

To resolve this issue, we can set the <a> element to display: block; or display: inline-block; so that it can behave as an isolated/independent block element.
